Fields

Name Type Description
"No." Code[20]

Specifies the unique identifier of the rebate, according to the specified number series.
Description Text[100]

Specifies the description of the rebate.
"Calculation Type" Enum "NICSOF Rebate Calc Type"

Specifies how the rebate is calculated. It can be a percentage of the sales amount, or a specific amount per sales or base unit of measure.
Value Decimal

Specifies the value of the rebate. If the rebate is calculated based on a percentage, this field will be a percentage that will be applied to the sales document line amount. If the rebate is calculated per unit of measure, this field will be an amount per quantity or quantity (base).
"Unit of Measure Code" Code[10]

Specifies the unit of measure code of the rebate. If the rebate is calculated based on a percentage, this field will be empty. If the rebate is calculated per unit of measure, this field will be required. A rebate will apply to a document line only if the unit of measure on the document line and the rebate matches.
"Rebate Posting Group" Code[20]

Specifies the general ledger accounts that will be used when you post rebate amounts.
"Value Type" Enum "NICSOF Rebate Value Type"

Specifies whether the rebate is calculated based on a specific value or tiered values based on the calculation type.
"Copied From Rebate No." Code[20]

Specifies the rebate number that this rebate was copied from.
"Rebate Category Code" Code[20]

Specifies the category that the rebate belongs to.
"Currency Code" Code[10]

Specifies the currency code of the rebate. A rebate will apply to a document only if the currency of the document and rebate matches. Leave this field empty to apply the rebate to local currency transactions.
"Starting Date" Date

Specifies the starting date of the rebate.
"Ending Date" Date

Specifies the ending date of the rebate.
"No. Series" Code[20]

Specifies the number series that was used to assign the number to this rebate.
Status Enum "NICSOF Rebate Status"

Specifies the status of the rebate. If the status is Draft, the rebate can be edited and rebate entries will not be calculated regardless of the start and end dates. If the status is Active, rebate entries will be calculated based on the start and end dates, and the rebate cannot be edited. If the status is Inactive, rebate entries will not be calculated regardless of the start and end dates and the rebate can be edited. This allows you to temporarily pause a rebate.
