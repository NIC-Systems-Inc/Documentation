Fields

Name Type Description
Type Enum "NICFCH API Type"

Specifies the type of API configuration.
"API Body" Blob

Specifies the body of the API request. This can be created using the 'Advanced Search' tool in FoodChain ID and pasted here.
"Item Template Code" Code[20]

Specifies the item template code to use when creating items.
"Use FCID Code As Item No." Boolean

Specifies whether to use the FoodChain ID code as the item number. If not enabled, the item number will be generated by the system.
"Send Zero-Costs" Boolean

Specifies whether to send items with zero cost to FoodChain ID.
"Use FCID Code As BOM No." Boolean

Specifies whether to use the FoodChain ID code as the production BOM number. If not enabled, the production BOM number will be generated by the system.
"Production BOM Version Nos." Code[20]

Specifies the number series that is used when creating production BOM versions. If left blank, you must manually define the number series on each production BOM.
"Replace Existing BOM if New" Boolean

Specifies whether to replace an existing production BOM with the new one from FoodChain ID. If not enabled, a new production BOM version will be created.
"BOM Version Starting Date Src" Enum "NICFCH BOM Starting Date Src"

Specifies how the starting date gets populated when a new production BOM version is created.
"BOM Line Quantity Source" Enum "NICFCH BOM Line Qty Source"

Specifies which FoodChain ID quantity is used when creating production BOM lines for ingredients.
"Create Item Specification" Boolean

Specifies whether to create an item specification when creating items.
"Use Pct. Comp. As Quantity" Boolean

Specifies whether to use the percentage composition as the quantity when creating non-packaging production BOM lines on recipes. In FoodChain ID, recipe quantities are defined per 100 grams. If this field is enabled, the quantities will be divided by 100. This does not apply to production BOMS created for finished goods.
