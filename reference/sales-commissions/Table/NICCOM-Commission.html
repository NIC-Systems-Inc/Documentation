Fields

Name Type Description
"No." Code[20]

Specifies the unique identifier of the commission, according to the specified number series.
Description Text[100]

Specifies the description of the commission.
"Calculation Type" Enum "NICCOM Commission Calc Type"

Specifies how the commission is calculated. It can be calculated as a percentage of sales amount, profit, or profit margin, or you can pay out an amount per unit of measure.
Value Decimal

Specifies the value of the commission. If the commission is calculated based on a percentage, this field will be a percentage. If the commission is calculated per unit of measure, this field will be an amount.
"Unit of Measure Code" Code[10]

Specifies the unit of measure code of the commission. If the commission is calculated based on a percentage, this field will be empty. If the commission is calculated per unit of measure, this field will be required. A commission will apply to a document line only if the unit of measure on the document line and the commission matches.
"Commission Posting Group" Code[20]

Specifies the general ledger accounts that will be used when you post commission amounts.
"Value Type" Enum "NICCOM Commission Value Type"

Specifies whether the commission is calculated based on a specific value or tiered values based on the calculation type.
"Copied From Commission No." Code[20]

Specifies the value of the Copied From Commission No. field
"Currency Code" Code[10]

Specifies the currency code of the commission. A commission will apply to a document only if the currency of the document and commission matches. Leave this field empty to apply the commission to local currency transactions.
"Starting Date" Date

Specifies the starting date of the commission.
"Ending Date" Date

Specifies the ending date of the commission.
"Commission Category Code" Code[20]

Specifies the category that the commission belongs to.
"Req Sales Doc Paid By Due Date" Option

Specifies if the commission will be paid only if the sales document is paid by the due date. This setting is ignored if 'Sales Document Is Fully Paid' is not enabled.
"Req Sales Doc Fully Paid" Option

Specifies if the commission will be paid only when the sales document has been fully paid.
"No. Series" Code[20]

Specifies the value of the No. Series field
Status Enum "NICCOM Commission Status"

Specifies the status of the commission. If the status is Draft, the commission can be edited and commission entries will not be calculated regardless of the start and end dates. If the status is Active, commission entries will be calculated based on the start and end dates, and the commission cannot be edited. If the status is Inactive, commission entries will not be calculated regardless of the start and end dates and the commission can be edited. This allows you to temporarily pause a commission.
