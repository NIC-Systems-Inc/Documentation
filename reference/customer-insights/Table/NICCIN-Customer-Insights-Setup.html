Properties

Name Value
Caption Customer Insights Setup

Fields

Name Type Description
"Primary Key" Code[10]

Specifies the value of the primary key.
Duration DateFormula

Specifies how far into the past to search for records to use in the customer insights calculation. You must enter a minus (-) and use one of the following time units: D, WD, W, M, Q, or Y.
"Date Range" Text[1024]

Specifies the default date range filter for the records on the Customer Insights page.
"Last Run Completed" DateTime

Specifies the date and the time at which last time calculation of the customer insights occurred.
"Early Payments Formula" DateFormula

Specifies the time period after the invoice posting date that an invoice is considered to be paid early when calculating the EARLY-PAYMENT metric. For example, 2W means a payment is early if it is fully paid within two weeks of the invoice posting date.
"Default Total Sales Target" Decimal

Specifies the default total sales amount a customer should achieve for the month. The total sales is the sales amount minus any discounts, refunds, rebates and taxes.
"Default Credit Sales Target" Decimal

Specifies the default credit sales amount a customer is expected to receive for the month.
"Default Invoice Sales Target" Decimal

Specifies the default invoice sales amount a customer should achieve for the month.
"Default Profit Target" Decimal

Specifies the default profit amount a customer should achieve for the month.
DefaultProfitMarginPctTarget Decimal

Specifies the default profit margin a customer should achieve for the month.
"Default Discount Target" Decimal

Specifies the default discount amount a customer is expected to receive for the month.
"Default Refund Target" Decimal

Specifies the default refund amount a customer is expected to receive for the month.
"Default Rebate Target" Decimal

Specifies the default rebate amount a customer is expected to receive for the month.
